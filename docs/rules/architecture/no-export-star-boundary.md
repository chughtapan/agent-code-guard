# `agent-code-guard/no-export-star-boundary`

**What it flags:** `export *` declarations in public-boundary or `index.ts`
barrel files.

**Why:** `export *` makes the boundary inherit *every current and future
export* from the target module. A helper added six months from now becomes
public API silently — no PR review, no CHANGELOG entry, no version bump
discussion. The boundary stops describing your contract and starts
mirroring whatever happens to be in the target file.

`export *` is also hostile to bundlers and IDE tooling: they can't
tree-shake or autocomplete a re-export they can't statically enumerate.

## Before (flagged)

```ts
// src/index.ts
export * from "./schema";
export * from "./runtime";
export * from "./db";
```

What does this package export? Nobody knows without reading three other
files. Adding `export const _internalCacheKey = "..."` to `./db` ships
that as public API.

## After (preferred) — named re-exports

```ts
// src/index.ts
export type { ProtocolFrame, ProtocolError } from "./schema";
export { parseFrame, encodeFrame } from "./schema";

export type { Runtime, RuntimeConfig } from "./runtime";
export { makeRuntime } from "./runtime";

export type { Database, DatabaseError } from "./db";
export { connect } from "./db";
```

Now the public surface is one file you can read top to bottom. Adding a
new export requires a deliberate edit to `index.ts`, which is where
CHANGELOG-worthy changes get reviewed.

## After (preferred) — selective re-export with renaming

When the underlying module has internal-vs-public distinctions, name the
public ones explicitly:

```ts
// src/index.ts
export { Frame as ProtocolFrame, parseFrame } from "./schema";
// _internalParser is NOT re-exported — it stays a schema/ implementation detail
```

## Options

This rule has no tunable options — `export *` on a boundary is structurally
wrong. Severity defaults to error.

## Suppressing per-file via a directive

For generated or codegen-output files where named re-exports would be
impractical (e.g., a barrel of 200 typed-ID brands generated from a
schema), suppress the rule for the file with a written reason:

```ts
// @agent-code-guard/architecture-exception: no-export-star-boundary
// reason: generated by `pnpm gen:ids`; see scripts/gen-ids.ts

export * from "./generated/ids";
```

The `reason:` line is required; missing reasons surface as
`architecture-directive-parse-error` diagnostics.

For hand-written barrels, the suppression smell is real — write the named
re-exports, even if it's tedious. Future you will thank present you when a
helper accidentally goes public.

## Rationale

`export *` is the difference between "this is my contract" and "whatever I
happen to have in this folder is my contract." The latter isn't a contract.
